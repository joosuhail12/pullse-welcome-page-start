Pullse Chat Widget Test Mode

This page demonstrates how to use the test mode feature of the Pullse Chat Widget. Test mode allows you to preview and test your chat widget configuration without sending real messages to your support system.

How to Enable Test Mode

There are several ways to enable test mode for the Pullse Chat Widget:

1. Using the data attribute in the script tag

<script src="https://cdn.pullse.io/embed.js" data-workspace-id="your-workspace-id" data-test-mode ></script>

2. Using the JavaScript API

<script> window.Pullse = window.Pullse || {}; window.Pullse.chat('init', { workspaceId: 'your-workspace-id', testMode: true }); </script>

3. Using the setTestMode command

<script> window.Pullse = window.Pullse || {}; window.Pullse.chat('init', { workspaceId: 'your-workspace-id' }); window.Pullse.chat('setTestMode', true); </script>

Test Mode Features

  • Visual indicators show that test mode is active
  • Simulated responses to your messages
  • Test file uploads without sending actual files
  • Preview all widget features without connecting to production systems
  • No real messages are sent to your support team

Use Cases for Test Mode

  • Preview the chat widget in your customer dashboard
  • Test different configurations before deploying to production
  • Demonstrate features to stakeholders
  • Train support staff on the chat interface
